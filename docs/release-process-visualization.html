ü§ñ Semantic Release Pipeline

Configuration: .releaserc.json

Semantic Release orchestrates version management, changelog generation, and file synchronization using a plugin-based architecture.

Plugin Execution Order

  1. @semantic-release/commit-analyzer
    Analyzes commit messages to determine version bump type using conventional commits
  2. @semantic-release/release-notes-generator
    Generates release notes from commits grouped by type (feat, fix, docs, etc.)
  3. @semantic-release/changelog
    Updates CHANGELOG.md with new release notes
  4. @semantic-release/exec
    Executes shell commands to update version in pyproject.toml and sync CLAUDE.md/README.md
  5. @semantic-release/github
    Creates GitHub release with distribution assets (dist/*)
  6. @semantic-release/git
    Commits version changes back to repository with [skip ci] flag

üì¶ PyPI Publishing (OIDC)

Token-Less Authentication

Using OpenID Connect (OIDC) trusted publishing, the workflow authenticates to PyPI without storing any API tokens. GitHub's identity provider issues short-lived tokens automatically.

Creates .publish.attestation PyPI->>GHA: Package published + attested

Attestation Strategy

Why Different Attestation Settings?

Test PyPI: attestations: false - Prevents creation of .publish.attestation files that would conflict with production publishing

Production PyPI: attestations: true - Generates Sigstore attestations for package authenticity verification

Repository URL Attestations Purpose
Test PyPI test.pypi.org/legacy/ Disabled Pre-release validation
Production PyPI pypi.org Enabled Official distribution

üîß Troubleshooting

Common Issues

Issue: No release created

Cause: No conventional commits since last release

Solution: Ensure commit messages use conventional format (feat:, fix:, etc.)

Issue: Attestation conflict error

Cause: Both PyPI publishes trying to create .publish.attestation

Solution: Already fixed - Test PyPI has attestations: false

Issue: Docker build timeout

Cause: Multi-platform builds are slow on cold cache

Solution: Wait ~5 minutes, cache will speed up subsequent builds

Issue: Self-assessment fails

Cause: agentready CLI not installed in workflow environment

Solution: Workflow installs with pip install -e . before running

Release Checklist

  1. Verify you're on main branch
    git branch --show-current should show "main"
  2. Make a conventional commit
    git commit -m "feat: trigger release to test new housekeeping features"
  3. Push to trigger workflow
    git push origin main
  4. Watch workflow execution
    Visit GitHub Actions ‚Üí Release
  5. Verify all distributions
    Check PyPI, Test PyPI, Docker Hub, GitHub Pages, and self-assessment

Expected Outcome

After ~3-5 minutes, you should see:

  • New version on PyPI and Test PyPI
  • Docker images on ghcr.io with new version tag
  • Updated documentation on GitHub Pages
  • Fresh self-assessment in examples/self-assessment/
  • Comprehensive release summary in workflow logs
